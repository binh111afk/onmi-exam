# Onmi Exam Design Language v1.0

> Design Philosophy

Onmi Exam is not an exam website.

Onmi Exam is a learning product.

Students should feel relaxed when opening the website, motivated while studying, and rewarded after finishing.

The interface must feel handcrafted by experienced product designers instead of AI-generated.

---

# Core Principles

1. Learning First

Everything exists to help users learn better.

Do not sacrifice usability for visual effects.

Reading speed is more important than visual complexity.

---

2. Calm UI

The UI should reduce stress.

Avoid visual overload.

Large whitespace.

Readable typography.

Soft shadows.

Neutral backgrounds.

---

3. Product, not Landing Page

Never design pages like startup landing pages.

Avoid giant hero sections.

Avoid huge marketing banners.

Avoid exaggerated call-to-action buttons.

Avoid "Buy Now" everywhere.

Users come here to study.

---

4. Invisible Design

Good design should almost disappear.

Users should focus on the content.

Not the interface.

---

5. Consistency

Every page must feel like it belongs to the same ecosystem.

Spacing.

Typography.

Motion.

Components.

Everything should feel consistent.

# AI Features

AI should feel like a tutor.

Never like ChatGPT pasted inside the website.

AI responses should

Be short

Friendly

Educational

Guide instead of giving answers immediately.

---

# Empty States

Every empty state should motivate.

Example

"You haven't started your Math journey yet."

Instead of

"No data found."

---

# Error States

Never blame users.

Offer clear recovery.

Example

"We couldn't load your exam.

Try again in a few seconds."

Instead of

"Unknown Error."

---

# Accessibility

Keyboard first.

Visible focus states.

High color contrast.

Touch targets >=44px.

Responsive.

---

# Performance

Animations under 200ms.

Avoid unnecessary JavaScript.

Prefer CSS animations.

Content loads before decoration.

Performance is a feature.

---

# AI Rules

Before generating UI, ask:

Does this improve learning?

Would Notion add this?

Would Apple simplify this?

Would Brilliant remove this?

If the answer is No,

remove it.

---

# Success Criteria

A first-time visitor should think

"This feels comfortable."

After studying

"I actually want to come back tomorrow."

Not

"Wow, cool animation."

The product should be remembered for helping people learn,
not for looking flashy.
